21.12 API 방식 통한 마이데이터 서비스 시행

- 은행, 보험사, 카드사 등 금융사별로 흩어진 개인신용정보를 모아 한 곳에서 통합 관리할 수 있는 서비스
(금융기관 거래이력, 인터넷 검색기록, 쇼핑몰 소비내역 등을 활용한 개인맞춤형 금융상품 추천 서비스)
(기존의 기업 중심의 데이터 활용 -> 개인 중심의 활용체계로 전환)
- 수취, 송금인 성명, 메모 등 적요 정보도 본인 조회 목적에 한해 서비스 범위에 포함될 수 있도록 허용됨
- 금융권 마이데이터(본인신용정보관리업) 사업, 12월 1일부터 전면 시행. 
- 고객은 여러 앱 설치할 필요 없이 하나의 은행 앱 또는 핀테크 앱 만으로 모든 은행의 계좌를 조회, 이체할 수 있게 됨
- 세계 최초로 시행하는 본격적인 금융 데이터 서비스


비대면 문화 확산과 디지털 전환 가속화 등과 맞물려 본격적인 금융 플랫폼 시대로의 전환을 촉발하는 계기가 될 것이 예상됨.




(영향)
1. 소비자 측면 - 분산되어 있는 신용정보 데이터를 한번에 확인, 관리 / 개인화, 맞춤화된 신 금융서비스 제공받을 수 있음 
2. 금융산업 측면 - 기존 대형 금융회사의 고객 데이터 독점 해소되면서 시장 지배력이 약화되고, 시장경쟁이 치열해짐에 따라 금융분야에서 개방형 혁신이 본격적으로 추진될 것으로 예상


<가이드라인>
신뢰받는 안전한 마이데이터 서비스 제공을 위해 기능적합성,보안취약점 점검 의무화
(기능적합성 심사: 신용정보법상 행위규칙 준수여부, 표준 API 규격적합성 등을 서비스 출시 전 확인
보안취약점 점검: 마이데이터 서비스 관련 시스템(응용 프로그램, DB, 웹서버, 정보보호시스템)에 대해 보안성 및 취약점 점검 실시)






고객데이터  개방
금융인프라 개방
결제 서비스 기능 개방 - 오픈뱅킹시스템 : 핀테크 기업이 은행 등의 금융기관과 개별적으로 계약하지 않고 은행의 조회, 이체 등의 기능을 표준화된 API 형태로 제공하고, 개발하는 서비스를 테스트를 할 수 있는 테스트베드 제공하는 공동결제 시스템. -> 개발 용이, 시간단축 가능


금융위, '오픈뱅킹 고도화 방안'에서 향후 오픈뱅킹에서 이용할 수 있는 대상 계좌를 예적금 계좌 등으로 확대하고, 참여기관은 원칙적으로 일정 수준의 데이터를 개방하는 것을 의무화하여 이용 정보의 범위도 확대할 계획을 밝힘. 그동안 이용기관으로만 참여하던 핀테크 기업도 은행의 조회기능에 해당하는 선불전자지급수단 고객 계정의 잔액, 거래내역 등을 제공하게 되며 계좌보유기관이 아닌 카드사의 경우 카드 보유내역, 청구금액 등이 개방할 예정.


금융사는 수십 년간 쌓아온 고객의 모든 데이터를 공개해야 하는 반면 빅테크·핀테크 기업이 보유한 비금융 정보는 쉽게 받아볼 수 없다며 ‘역차별’ 문제를 제기. 오픈마켓 계열사 있는 마이데이터 사업자의 경우 통합 조회 통해 고객에게 제공할 수 있는 정보의 범위가 은행과 비교해 훨씬 많고 상세할 수 밖에 없고, 데이터 활용한 다양한 서비스 제공에서도 훨씬 유리하게 작용해 불평등한 경쟁 이뤄질 수 있음.
한편, 빅테크 업체는 마이데이터 운영 가이드라인이 적요 정보, 상세정보는 선택 동의 통해서만 정보 제공하게 하고, 정기전송 요구도 주1회로 제한하고 있다는 점 문제 삼음 - 사용자의 금융 데이터를 실시간으로 숮비하고 패턴 분석을 통해 얼마나 잘 컨설팅해주느냐가 경쟁력 좌우하는데, 주 1회 신용 정보가 제공된다면  실시간 데이터를 기반으로 한 자산관리 서비스나 지출 분석 서비스 등은 아예 불가능
=> 마이데이터 사업이 '무늬만 혁신'에 그칠 가능성 적잖음.






금융 - 데이터 - IT를 융합한 디지털 금융 혁신이 가속화
<마이데이터 도입의 배경과 영향, 도입 이후 소비자 정보 보호할 수 있는 방안>

(배경)
: 개인이 본인 데이터 활용해 자산을 통합적으로 관리하기 어렵다. 금융기관 역시 고객들의 거래, 지출, 투자내역을 각자 축적하고 분석해 자체 활용하지만, 타 금융기관 이용내역 확인할 수 없기 때문에 고객 중심의 통합 자산관리 서비스 제공하는 데 한계 지님. 
금융산업 내 비경쟁적 시장 환경을 조장. 대형 금융기관은 방대한 데이터를 축적해 이를 영업에 활용할 수 있지만, 중소형 금융기관이나 신규 진입 업체들은 그렇지 못해 정보 비대칭 상황이 나타남. 그 결과 대형 금융기관은 시장 지배력을 공고하게 유지하나 여타 기관들은 경쟁 열위에 놓이게 된다.
이러한 문제 인식 하에 마이데이터 개념이 등장. 

(영향)
1. 소비자 측면 - 분산되어 있는 신용정보 데이터를 한번에 확인, 관리 -> 신속한 의사결정 가능, 효율적인 자금관리, 이를 통한 예금이나 투자 수익 증대 가능성 / 개인화, 맞춤화된 금융서비스 제공받을 수 있음 -> 만족도 증가
 
2. 금융산업 측면 - 기존 대형 금융회사의 고객 데이터 독점 해소되면서 시장 지배력이 약화될 것(소비자가 제3자의 서비스를 통해 결제계좌의 잔액과 거래내역을 쉽게 파악하고 지급지시 서비스를 통해 간접적으로 이체나 송금 업무까지도 수행할 수 있음) / 시장경쟁이 치열해짐에 따라, 정보 제공으로 인해 금융분야에서 개방형 혁신이 본격적으로 추진될 것으로 예상 / 고객의 소비패턴과 재무현황,위험성향 등에 대해 종합적인 분석이 이루어지면 이를 바탕으로 더욱 정교한 금융상품을 만들 수 있음(ex. 기존에 파악하지 못했던 재무건전성, 소비행태, 납세정보 파악 가능. 이를 토대로 상환능력 예측하는 새로운 지표 발굴해 대출심사에 활용 가능)


(소비자정보 보호 방안)
1. 제3자로의 데이터 전송과정에서 확실한 인증 및 보안요건 규정
2. 제3자의 데이터 보관 및 활용에 있어서의 높은 수준의 규제
3. 소비자의 자기결정권 보장
